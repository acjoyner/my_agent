# Feature Specification: Resume Auto-Updater

**Feature Branch**: `006-resume-auto-updater`  
**Created**: 2026-04-30  
**Status**: Draft  
**Input**: User description: "Add a skill to update my resume with projects from GitHub (current/complete). Document formats: Word (.docx) and PDF. Save to Google Drive. User review is required before making changes."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - GitHub Project Sync (Priority: P1)
As a developer, I want the agent to fetch my most recent GitHub repositories and descriptions so that I can keep my "Projects" section of my resume up to date without manual tracking.

**Why this priority**: Core data source for the resume updates.

**Independent Test**: Can be tested by running the `fetch_github_projects` tool and verifying it returns a list of repo names, descriptions, and primary technologies.

**Acceptance Scenarios**:
1. **Given** a valid GitHub username, **When** requested, **Then** the system returns the top 3-5 most recently active repositories.

---

### User Story 2 - Document Modification & Review (Priority: P1)
As a user, I want to see the proposed new bullet points for my resume before they are written to the file so that I can ensure the phrasing matches my professional voice.

**Why this priority**: High-stakes modification; requires a human-in-the-loop review gate.

**Independent Test**: Can be tested by initiating an update and verifying the agent presents a text diff/proposal and waits for "Approve" or "Edit" input.

**Acceptance Scenarios**:
1. **Given** new project data, **When** processed, **Then** the agent outputs: "I propose adding these bullets: [X]. Do you approve?"
2. **Given** user approval, **When** finalized, **Then** the `.docx` file in `uploads/` is modified.

---

### User Story 3 - Google Drive Sync (Priority: P2)
As a user, I want the final versions of my updated resume (Word and PDF) to be automatically backed up to my Google Drive so that they are accessible anywhere.

**Why this priority**: Ensures the updated document is safely stored and distributable.

**Independent Test**: Can be tested by finishing a resume update and verifying a new file version appears in the user's Google Drive.

**Acceptance Scenarios**:
1. **Given** a modified resume file, **When** sync is triggered, **Then** the system uses the Google Drive API to upload the file to a designated "Resumes" folder.

---

### User Story 4 - Web App Synchronization (Priority: P2)
As a user, I want my live 'resume-app' on Firebase to be updated with my latest projects so that my online presence is always in sync with my physical document.

**Why this priority**: Enhances the user's online brand and automates a multi-step deployment process.

**Independent Test**: Can be tested by verifying that the `Projects.tsx` file in the `resume-app` directory is updated and the Firebase deployment command is triggered.

**Acceptance Scenarios**:
1. **Given** approved project updates, **When** web sync is triggered, **Then** the React source code in the `resume-app` project is updated and `firebase deploy` is executed.

## Requirements *(mandatory)*

### Functional Requirements
- **FR-001**: System MUST use the GitHub API to fetch public/private repository details.
- **FR-002**: System MUST use `python-docx` to programmatically update the `.docx` resume in the `uploads/` directory.
- **FR-003**: System MUST convert the updated Word document to PDF format.
- **FR-004**: System MUST implement a "Review Gate" logic where changes are staged but not committed until user confirmation.
- **FR-005**: System MUST utilize the existing Google Drive tools to upload the final artifacts.
- **FR-006**: System MUST update the React constants in the `resume-app` project (at `/Users/anthonyjoyner/Documents/Projects/resume/resume-app/client/src/pages/Projects.tsx`) and execute `firebase deploy`.

### Key Entities
- **Project Record**: Data fetched from GitHub (Name, Description, Tech Stack).
- **Resume Proposal**: A draft of the new bullet points generated by the AI.
- **Resume Artifact**: The final .docx, .pdf, and live Web App.

## Success Criteria *(mandatory)*

### Measurable Outcomes
- **SC-001**: Resume update proposal generated in under 10 seconds.
- **SC-002**: 100% formatting preservation (margins, fonts) when updating the .docx file.
- **SC-003**: Successful upload to Google Drive verified with a 200 OK API response.

## Assumptions
- The user has a GitHub Personal Access Token (or uses public fetching).
- The `uploads/` directory contains a document with a clear "Projects" or "Work Experience" heading.
- Google Drive OAuth credentials are already configured in the project.
